Canvas Layering

Game introduction example for your assessment task

What is Frogger?

In Frogger, players control a frog who must cross a busy road and a hazardous river to reach its home on the other side. It sounds simple, but the path is filled with dangers! This game captivates players with its simple yet challenging gameplay.

Gameplay Mechanics

The core gameplay of Frogger involves navigating the frog, one hop at a time, through multiple lanes of traffic and a river teeming with logs and turtles.

Controls

The frog can be moved in four directions:

  • Up: Hops forward into the next lane or onto an object in the river.
  • Down: Hops backward.
  • Left: Hops to the left.
  • Right: Hops to the right.

Each hop moves the frog a fixed distance.

Objective

The primary goal is to guide five frogs, one by one, to their homes (lily pads or logs) at the top of the screen. You must avoid:

  • On the Road: Cars, trucks, and buses. Getting hit means losing a life.
  • In the River: Drowning by falling into the water. You must land on logs or the backs of turtles. Be careful, as some turtles may submerge!
  • Running out of time for each frog.

Bonus points can be earned by collecting flies that occasionally appear in the homes or by escorting a lady frog.

<div class="wrapper"> ... </div>

This div element acts as a container for our game canvases. The class wrapper will be used in style.css to style this container (e.g., set its size, position, and background).

<canvas id="canvas1"></canvas> (and canvas2, canvas3, canvas4)

These are HTML5 <canvas> elements. We use JavaScript to draw graphics on these canvases. Each canvas has a unique id so we can target it with JavaScript. They are layered on top of each other using CSS to create the game's visual depth.

<script type="module" src="app.js"></script>

This line includes our main JavaScript file, app.js. The type="module" attribute is important because it allows us to use modern JavaScript features like import and export to organise our code into multiple files (modules). This script tag is usually placed at the end of the body so that the HTML structure is loaded before the script tries to interact with it.

This game uses multiple HTML <canvas> elements layered on top of each other. Think of them as transparent sheets.

  • canvas1 (ctx1): Used for logs, turtles, and water ripples. This is behind the frog.
  • canvas2 (ctx2): Used for the road background and cars. This is also behind the frog.
  • canvas3 (ctx3): Used for the Frogger character and dust particles from its movement. This is the "main character" layer.
  • canvas4 (ctx4): Used for the grass foreground and the scoreboard. This is on top of everything.
This layering allows different game elements to be drawn and cleared independently without affecting others.

.wrapper { ... }

This CSS rule targets the HTML element with the class wrapper (our main game container).

background: url(' . /img/background-water.png'): Sets a background image for the wrapper. This image will be visible behind the canvases if they are transparent or don't cover the entire area.

position: absolute: This allows us to position the wrapper precisely on the page.

width: 600px and height: 600px: Defines the dimensions of our game area.

top: 50% and left: 50%: Positions the top-left corner of the wrapper at the center of the page.

translate: -50% -50%: This is a common trick used with top: 50% and left: 50% to truly center an element. It shifts the element back by half of its own width and height, ensuring its center aligns with the page's center.

border: 5px solid: Adds a 5-pixel wide solid border around the game area.

canvas { ... }

This CSS rule targets all <canvas> elements on the page.

width: 100% and height: 100%: Makes each canvas take up the full width and height of its parent container (the .wrapper div). So, each canvas will be 600px by 600px.

position: absolute: This is key for layering. It takes the canvases out of the normal document flow.

top: 0 and left: 0: Positions each canvas at the top-left corner of their parent (the .wrapper). Since they all have this, they will stack directly on top of each other.

collision( )

export function collision(first, second) { ... }: A utility function to detect collision between two rectangular objects (first and second). Each object is expected to have x, y, width, and height properties.

return !( ... ): The function returns true if there is a collision, and false otherwise. The logic checks for conditions where the two rectangles do *not* overlap. If none of these "no overlap" conditions are met, it means they must be overlapping.

first.x > second.x + second.width: True if first is entirely to the right of second.

first.x + first.width < second.x: True if first is entirely to the left of second.

first.y > second.y + second.height: True if first is entirely below second.

first.y + first.height < second.y: True if first is entirely above second.

If any of these are true, there's no collision. The ! (NOT operator) inverts this, so it returns true only when all these "no overlap" conditions are false (i.e., there is an overlap).

2. Player Character ( ./scripts/frogger.js )

This file defines the Frogger class, which controls the player character's behavior, appearance, and movement.

import { canvas3, ctx3, keys, scored } from '../app.js'
import { grid } from './obstacles.js'

const froggerSprite = new Image()
froggerSprite.src = 'img/frog-spritesheet.png'

export class Frogger {
  constructor() {
    this.spriteWidth = 250
    this.spriteHeight = 250
    this.width = this.spriteWidth / 5
    this.height = this.spriteHeight / 5
    this.x = canvas3.width / 2 - this.width / 2
    this.y = canvas3.height - this.height - 40
    this.moving = false
    this.frameX = 0
    this.frameY = 0
  }
  update() {
    if (keys[0] === 'UP') {
      if (this.moving === false) {
        this.y -= grid
        this.moving = true
        this.frameY = 0
      }
    }
    if (keys[0] === 'DOWN') {
      if (
        this.y < canvas3.height - this.height * 2
        && this.moving === false
      ) {
        this.y += grid
        this.moving = true
        this.frameY = 3
      }
    }
    if (keys[0] === 'LEFT') {
      if (this.x > this.width && this.moving === false) {
        this.x -= grid
        this.moving = true
        this.frameY = 2
      }
    }
    if (keys[0] === 'RIGHT') {
      if (
        this.x < canvas3.width - this.width * 2
        && this.moving === false
      ) {
        this.x += grid
        this.moving = true
        this.frameY = 1
      }
    }
    if (this.y < 0) scored()
  }
  draw() {
    ctx3.drawImage(
      froggerSprite,
      this.frameX * this.spriteWidth,
      this.frameY * this.spriteHeight,
      this.spriteWidth,
      this.spriteHeight,
      this.x - 25,
      this.y - 25,
      this.width * 2,
      this.height * 2
    )
  }
  jump() {
    if (this.moving === false) this.frameX = 1
    else if (this.frameX === 1) this.frameX = 0
  }
}

Frogger Class

export class Frogger { ... }: Defines a class named Frogger. The export keyword allows this class to be imported and used in other files (like app.js).

constructor( )

The constructor is a special method for creating and initializing an object created with a class.

this.spriteWidth = 250 and this.spriteHeight = 250: These define the width and height of a single frame (one frog pose) within the froggerSprite.png image.

this.width = this.spriteWidth / 5 and this.height = this.spriteHeight / 5: These calculate the actual width and height the frog will have on the game canvas. Here, the sprite is scaled down by a factor of 5. So, the frog will be 50x50 pixels logically.

this.x = canvas3.width / 2 - this.width / 2: Sets the initial horizontal (x) position of the frog to the center of canvas3.

this.y = canvas3.height - this.height - 40: Sets the initial vertical (y) position of the frog near the bottom of canvas3 (40 pixels from the very bottom edge, after accounting for its height).

this.moving = false: A boolean flag to track if the frog is currently in the middle of a move. This prevents multiple moves from happening simultaneously.

this.frameX = 0; and this.frameY = 0: These will be used to select which frame from the spritesheet to draw. frameX typically cycles through animation frames (e.g., for a jump), and frameY selects the row in the spritesheet corresponding to the frog's direction (up, down, left, right).

update( )

This method is called in each frame of the game loop (from app.js) to update the frog's state, primarily its position based on key presses.

if (keys[0] === 'UP') { ... }: Checks if the most recently pressed key (stored at keys[0]) is 'UP'.

if (this.moving === false) { ... }: Ensures the frog only moves if it's not already in the process of moving.

this.y -= grid: Moves the frog up by one grid unit. (Y decreases as you go up in canvas coordinates).

this.moving = true: Sets the flag to indicate the frog is now moving.

this.frameY = 0: Sets frameY to 0, which should correspond to the row in the spritesheet showing the frog facing upwards.

Similar blocks handle 'DOWN', 'LEFT', and 'RIGHT' movements:

For 'DOWN': if (this.y < canvas3.height - this.height * 2 && this.moving === false) { ... } This includes a boundary check: this.y < canvas3.height - this.height * 2 ensures the frog doesn't move off the bottom of the screen (or too far down). this.frameY = 3 for the downward-facing sprite.

For 'LEFT': if (this.x > this.width && this.moving === false) { ... } Boundary check: this.x > this.width ensures the frog doesn't move too far left (allowing some of its width to be visible). this.frameY = 2 for the left-facing sprite.

For 'RIGHT': if (this.x < canvas3.width - this.width * 2 && this.moving === false) { ... } Boundary check: this.x < canvas3.width - this.width * 2 ensures the frog doesn't move too far right. this.frameY = 1 for the right-facing sprite.

if (this.y < 0) scored( );: If the frog's y-position goes above the top of the canvas (y < 0), it means the frog has successfully crossed. The scored( ) function (imported from app.js) is then called.

draw( )

This method is responsible for drawing the frog on the canvas (ctx3).

ctx3.drawImage(...): This is the core canvas drawing command for images. It takes several arguments:

froggerSprite: The image object (our spritesheet) to draw from.

this.frameX * this.spriteWidth: The x-coordinate of the top-left corner of the sub-rectangle (the specific frame) to cut from the spritesheet.

this.frameY * this.spriteHeight: The y-coordinate of the top-left corner of the sub-rectangle from the spritesheet.

this.spriteWidth: The width of the sub-rectangle (one frame) from the spritesheet.

this.spriteHeight: The height of the sub-rectangle (one frame) from the spritesheet.

this.x - 25: The x-coordinate on the canvas where the top-left corner of the drawn image should be placed. The -25 is an offset. Since the drawn image (this.width * 2, this.height * 2) is larger than the frog's logical size, this offset helps center the larger visual representation around the frog's logical x, y point.

this.y - 25: The y-coordinate on the canvas, with a similar offset.

this.width * 2: The width the image should be drawn on the canvas. Here, it's drawn at twice its logical width (50px * 2 = 100px).

this.height * 2: The height the image should be drawn on the canvas (twice its logical height).

This means the frog's logical hitbox might be 50x50, but it's drawn as a 100x100 image, which can make it visually more appealing or easier to see.

jump( )

This method is called when a movement key is pressed (from app.js). It controls the frog's jumping animation frame.

if (this.moving === false) this.frameX = 1: If the frog is not currently moving (i.e., it's about to start a jump), set this.frameX to 1. This likely selects the "jump" or "mid-move" frame from the spritesheet row determined by this.frameY.

      }

import { ctx1, ctx3, keys } from '../app.js'
import { frogger } from '../app.js'

const particlesArray = []
const maxParticles = 300
export const ripplesArray = []

export class Particle {
  constructor(x, y) {
    this.x = x + 25
    this.y = y + 25
    this.radius = Math.random() * 20 + 1
    this.opacity = 1
    this.directionX = Math.random() - 0.5
    this.directionY = Math.random() - 0.5
  }
  update() {
    this.x += this.directionX
    this.y += this.directionY
    if (this.opacity > 0.1) {
      this.opacity -= 0.9
    }
    if (this.radius > 0.15) {
      this.radius -= 0.14
    }
  }
  draw() {
    ctx3.fillStyle = `rgb(150 150 150 / ${this.opacity})`
    ctx3.beginPath()
    ctx3.arc(this.x, this.y, this.radius, 0, Math.PI * 2)
    ctx3.fill()
    ctx3.closePath()
  }
  ripple() {
    if (this.radius < 50) {
      this.radius += 0.7
      this.x -= 0.03
      this.y -= 0.03
    }
    if (this.opacity > 0) {
      this.opacity -= 0.02
    }
  }
  drawRipple() {
    ctx1.strokeStyle = `rgb(255 255 255 / ${this.opacity})`
    ctx1.beginPath()
    ctx1.arc(this.x, this.y, this.radius, 0, Math.PI * 2)
    ctx1.stroke()
    ctx1.closePath()
  }
}

export function handleParticles() {
  // dust
  for (let i = 0; i < particlesArray.length; i++) {
    particlesArray[i].update()
    particlesArray[i].draw()

    for (let i = 0; i < 30; i++) {
      particlesArray.pop()
    }
  }
  if (
    keys.length > 0 && frogger.y > 250 &&
    particlesArray.length < maxParticles + 10

  }
}
